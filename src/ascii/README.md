# ASCII 转换模块

本目录封装图片转 ASCII 字符画的核心逻辑，主线程只负责图片解码、Canvas 缩放和 UI 状态，真正的字符画转换放在 Web Worker 中执行，避免大图转换时阻塞页面。

## 模块内容

- `types.ts`：定义模式、精度、字符集、图片帧、转换结果和 Worker 消息结构。
- `constants.ts`：保存默认参数、字符集、精度档案、文件大小和列数限制。
- `convert.ts`：实现灰度明暗映射、透明像素白底合成、Sobel 轮廓检测、字符选择和最终转换入口。
- `browserImage.ts`：在浏览器主线程中读取图片尺寸，并把图片缩放为目标采样像素帧。
- `convert.worker.ts`：Worker 入口，接收像素帧和参数，调用核心转换函数。
- `workerClient.ts`：主线程 Worker 客户端，负责启动、取消和接收转换任务。

## 转换策略

灰度明暗模式会把图片缩放到目标字符尺寸，然后用 `0.2126 * R + 0.7152 * G + 0.0722 * B` 计算亮度。字符集按从浅到深排序，亮像素映射到空格或低密度字符，暗像素映射到高密度字符。反色模式会反转密度映射，以适配深色背景。

轮廓线稿模式会先生成亮度网格，再按精度档案决定是否做 3x3 轻量模糊。随后使用 Sobel 算子计算边缘强度和方向，强度低于阈值的采样点输出空格，强度足够高的采样点根据方向输出 `-`、`|`、`/`、`\`。

## 尺寸与性能限制

目标列数由用户选择，范围为 40-240。实际行数按 `原图高度 / 原图宽度 * 列数 * 0.5` 计算，这个 0.5 是等宽字符宽高修正系数，用来抵消字符格通常比字符本身更高的问题。

首版限制输入图片最大 20 MB，转换前会先把图片缩放到目标采样尺寸，不直接处理原始全尺寸像素。参数变化会经过 UI debounce，并且旧 Worker 会被取消，避免连续拖动滑块时堆积转换任务。
